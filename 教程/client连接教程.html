OpenVPN DNS分流完整教程

1. 清理无效文件

清理配置过程中创建的临时文件（如果存在）：

del \"%USERPROFILE%\\\\setup_dns_redirect.ps1\"

2. 系统要求与准备

  • Windows 10/11 系统
  • OpenVPN 客户端已安装
  • 管理员权限（重要）
  • 确保以下文件存在于 OpenVPN config 目录中：
    • overture-windows-amd64.exe - DNS分流核心程序
    • config.json - DNS分流配置文件
    • client.ovpn - OpenVPN配置文件
    • client_pre.bat - 连接前执行脚本
    • client_down.bat - 断开后执行脚本

3. 端口转发配置

在管理员权限的命令提示符或PowerShell中运行以下命令：

netsh interface portproxy add v4tov4 listenport=53 listenaddress=127.0.0.1 connectport=1054 connectaddress=127.0.0.1

验证端口转发是否配置成功：

netsh interface portproxy show all

4. 配置文件说明

client.ovpn 配置文件关键设置：

script-security 2 up \"./client_pre.bat\" down \"./client_down.bat\" dhcp-option DNS 127.0.0.1 redirect-gateway def1

client_pre.bat 脚本内容：

@echo off start /min \"\" \"%~dp0overture-windows-amd64.exe\" -c \"%~dp0config.json\" >nul 2>&1 exit 0

client_down.bat 脚本内容：

@echo off taskkill /f /im overture-windows-amd64.exe >nul 2>&1 exit 0

5. 连接步骤

  1. 确保端口转发已配置（第3步）
  2. 以管理员身份启动OpenVPN GUI
  3. 连接VPN配置文件
  4. 等待连接成功建立
  5. 确认overture服务已自动启动

6. 验证DNS分流效果

连接成功后，可以使用以下方法验证：

  • 访问国内外网站，检查访问速度和稳定性
  • 使用在线IP检测网站验证IP地址是否为VPN服务器IP
  • 查看overture控制台输出（如果可见），确认其处理不同域名的DNS查询

7. 常见问题与解决方案

问题1: 连接失败

解决方案：检查脚本输出重定向是否正确，确保脚本最后有 exit 0 命令

问题2: DNS分流无效

解决方案：确认端口转发配置正确，overture服务正常运行，DNS缓存已清除

问题3: 无法启动overture

解决方案：检查config.json配置文件是否正确，确保端口1054未被占用

注意事项：
  • 所有操作需要管理员权限
  • 端口转发配置后需要重启生效（如果配置失败）
  • 确保config.json中的DNS服务器地址有效
  • 如果修改配置文件，需要重新连接VPN才能生效
成功标志：
  • VPN连接成功建立
  • IP地址显示为VPN服务器IP
  • 国内外网站访问正常
  • overture服务在端口1054监听
