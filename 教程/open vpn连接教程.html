OpenVPN DNS分流优化教程

1. 系统要求与准备

  • Windows 10/11 系统
  • OpenVPN 客户端已安装
  • 管理员权限（重要）
  • 确保以下文件存在于 OpenVPN config 目录中：
    • overture-windows-amd64.exe - DNS分流核心程序
    • config.json - DNS分流配置文件
    • client.ovpn - OpenVPN配置文件
    • client_pre.bat - 连接前执行脚本
    • client_down.bat - 断开后执行脚本

2. 端口转发配置（仅需设置一次）

在管理员权限的命令提示符或PowerShell中运行以下命令：

netsh interface portproxy add v4tov4 listenport=53 listenaddress=127.0.0.1 connectport=1054 connectaddress=127.0.0.1

验证端口转发是否配置成功：

netsh interface portproxy show all
注意：如果重启计算机，需要重新配置端口转发（或创建开机启动脚本）

3. 配置文件说明

client_pre.bat 脚本内容（连接前执行）：

@echo off start /min \"\" \"%~dp0overture-windows-amd64.exe\" -c \"%~dp0config.json\" >nul 2>&1 exit 0

client_down.bat 脚本内容（断开后执行）：

@echo off taskkill /f /im overture-windows-amd64.exe >nul 2>&1 exit 0

4. 操作流程

常规连接流程（日常使用）：

  1. 以管理员身份启动OpenVPN GUI
  2. 连接 [client.ovpn](file:///c:/Program%20Files/OpenVPN/config/client.ovpn) 配置
  3. 系统自动执行 [client_pre.bat](file:///c:/Program%20Files/OpenVPN/config/client_pre.bat)，启动 [overture-windows-amd64.exe](file:///c:/Program%20Files/OpenVPN/config/overture-windows-amd64.exe)
  4. 验证连接成功（IP地址、网站访问）

首次配置流程：

  1. 确保所有必要文件存在
  2. 以管理员权限配置端口转发（仅需一次）
  3. 验证配置文件正确
  4. 测试连接

重启电脑后流程：

  1. 重新配置端口转发（如果未创建开机脚本）
  2. 以管理员身份启动OpenVPN GUI
  3. 连接 [client.ovpn](file:///c:/Program%20Files/OpenVPN/config/client.ovpn) 配置

断开连接流程：

  1. 在OpenVPN GUI中断开连接
  2. 系统自动执行 [client_down.bat](file:///c:/Program%20Files/OpenVPN/config/client_down.bat)，停止 [overture-windows-amd64.exe](file:///c:/Program%20Files/OpenVPN/config/overture-windows-amd64.exe)

5. 故障排除

连接失败：

  • 确认以管理员权限运行OpenVPN
  • 检查 [client_pre.bat](file:///c:/Program%20Files/OpenVPN/config/client_pre.bat) 和 [client_down.bat](file:///c:/Program%20Files/OpenVPN/config/client_down.bat) 脚本输出重定向是否正确
  • 确保脚本最后有 exit 0 命令

DNS分流无效：

  • 确认端口转发配置正确
  • 检查 [overture-windows-amd64.exe](file:///c:/Program%20Files/OpenVPN/config/overture-windows-amd64.exe) 是否在运行（端口1054监听）
  • 清除DNS缓存：ipconfig /flushdns

无法启动overture：

  • 检查 [config.json](file:///c:/Program%20Files/OpenVPN/config/config.json) 配置文件是否正确
  • 确认端口1054未被占用
  • 检查 [overture-windows-amd64.exe](file:///c:/Program%20Files/OpenVPN/config/overture-windows-amd64.exe) 文件是否存在

6. 高级配置（可选）

创建开机自动配置端口转发脚本：

创建 [startup_portproxy.bat](file:///c:/Program%20Files/OpenVPN/config/startup_portproxy.bat) 文件：

@echo off netsh interface portproxy add v4tov4 listenport=53 listenaddress=127.0.0.1 connectport=1054 connectaddress=127.0.0.1 echo 端口转发配置完成 pause

将此脚本添加到Windows启动项中以实现开机自动配置

注意事项：
  • 所有操作需要管理员权限
  • 端口转发配置后可能需要重启生效
  • 确保 [config.json](file:///c:/Program%20Files/OpenVPN/config/config.json) 中的DNS服务器地址有效
  • 如果修改配置文件，需要重新连接VPN才能生效
  • 重启电脑后需要重新配置端口转发（除非创建开机脚本）
成功标志：
  • VPN连接成功建立
  • IP地址显示为VPN服务器IP
  • 国内外网站访问正常
  • [overture-windows-amd64.exe](file:///c:/Program%20Files/OpenVPN/config/overture-windows-amd64.exe) 服务在端口1054监听
  • 显示 \"Overture is listening on :1054\"
小贴士：
  • [overture-windows-amd64.exe](file:///c:/Program%20Files/OpenVPN/config/overture-windows-amd64.exe) 会根据 [config.json](file:///c:/Program%20Files/OpenVPN/config/config.json) 配置自动分流国内外DNS查询
  • 国内域名使用DNSPod（119.29.29.29）解析
  • 国外域名使用OpenDNS（208.67.222.222）解析
  • 连接时 [client_pre.bat](file:///c:/Program%20Files/OpenVPN/config/client_pre.bat) 自动启动服务，断开时 [client_down.bat](file:///c:/Program%20Files/OpenVPN/config/client_down.bat) 自动停止服务
