部署与操作步骤

  1. Fork仓库并开启GitHub Actions Workflow
    • 访问 项目仓库，点击“Fork”按钮，将仓库复制到你的GitHub账号下。
    • 进入Fork后的仓库，点击“Actions”标签，启用所有Workflow（Collect、Refresh/Process、Checkin）。默认每6小时自动运行更新节点。
    • 注意：视频中可能提及“Process”，而文字提及“Refresh”，请以仓库实际Workflow名称为准（视频优先）。手动运行一次以确认生效。
  2. 创建GitHub Gist
    • 访问 https://gist.github.com，创建一个新的Gist（内容随意，如“test”）。
    • 保存后，获取Gist URL（如 https://gist.github.com/username/xxxxxxxxxxxxxxxx），记录 username/gist_id（如 username/xxxxxxxxxxxxxxxx）。
    • 部署提示：确保Gist为公共可见，Actions将自动更新其内容为节点列表。
  3. 生成Personal Access Token (PAT)
    • 访问 GitHub Tokens设置页面，点击“Generate new token”。
    • 设置名称（如“VPN-Gist”），选择较长有效期（如90天），授予“Gists”和“repo”读写权限。
    • 复制生成的PAT，妥善保存，切勿泄露。
  4. 设置环境变量
    • 在Fork仓库的“Settings” > “Secrets and variables” > “Actions” > “Secrets”中，添加以下变量：
      • GIST_LINK：值为第2步的 username/gist_id。
      • GIST_PAT：值为第3步的PAT。
    • 部署提示：检查仓库README，确认是否需要额外变量（如订阅源URL）。
  5. 启用并监控GitHub Actions
    • 返回“Actions”页面，确认Workflow已启用。默认每6小时（或2小时，视仓库配置）更新一次。
    • 查看运行日志，检查Gist是否更新为最新节点列表（如vless://...或订阅URL）。
    • 重要：若停止使用，禁用Workflow或Actions，避免资源浪费。
  6. 导入节点到客户端
    • 打开第2步创建的Gist，复制节点链接或订阅地址（如 订阅二维码）。
    • 在v2rayN/Clash中导入：
      • 手动导入：粘贴节点链接。
      • 订阅导入：使用Gist中的订阅URL或二维码，设置自动更新（建议每小时）。
    • 测试连接，确保YouTube、Netflix等可访问。选择高可用节点（如美国、日本）以提升速度。

常见问题与解决方案

  • 节点无法连接：检查节点是否过期，重新运行Workflow，或切换其他节点（高峰期可能不稳定）。
  • Workflow未自动更新：确认Workflow名称（Process/Refresh），检查GIST_LINK和GIST_PAT，或查看Actions日志。
  • Gist未更新：确保PAT授予Gists和repo权限，手动运行Workflow验证。
