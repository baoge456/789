GitHub 代码推送完整教程

使用个人访问令牌(Personal Access Token)进行身份验证

创建个人访问令牌

1
登录GitHub → 点击右上角头像 → Settings
2
左侧菜单选择 Developer settings → Personal access tokens → Tokens (classic)
3
点击 "Generate new token"，选择 "Classic"
4
给token起个名字，设置过期时间(推荐选择 长期 )，并勾选以下权限：
  • repo - 完全控制仓库
  • workflow - 更新工作流文件
5
点击 "Generate token" 并立即复制token（只会显示一次）

重要提示：Token只会显示一次，请务必立即保存。如果丢失，需要重新生成。

推送代码完整流程

1. 初始化Git仓库

打开命令行，进入项目目录：

git init

2. 添加远程仓库地址

将GitHub仓库添加为远程仓库：

git remote add origin https://github.com/用户名/仓库名.git

3. 添加文件到暂存区

git add .

4. 提交更改

git commit -m "提交描述"

5. 重命名分支（可选）

git branch -M main

6. 使用Token推送代码

运行推送命令：

git push -u origin main

当系统提示输入密码时，粘贴您的Token（而不是GitHub密码）

替代方法：您也可以将Token直接嵌入远程URL中：

git remote set-url origin https://您的TOKEN@github.com/用户名/仓库名.git

设置后，后续推送不再需要输入凭据。

常见问题与解决方案

权限被拒绝 (403错误)

解决方案：

  • 检查Windows凭据管理器，删除旧的GitHub凭据
  • 确保Token具有repo和workflow权限

缺少workflow权限

解决方案：

  • 重新生成Token并确保勾选workflow权限

不是Git仓库

解决方案：

  • 运行git init初始化仓库

日常开发流程

完成初始设置后，日常开发流程简化为：

git add .
git commit -m "描述更改"
git push
